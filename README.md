# http_HTTPСервер

Расширение конфигурации 1С (cfe), стандартизирующее HTTP-сервисы для прикладных подсистем БСП-конфигураций. Целевая платформа — 8.3.27, целевая БСП — УТ 11.5.

## Возможности

- **HTTP-конвейер** (`http_HTTPКонвейер`) — pipeline обработки: CORS → OPTIONS → Auth → авторизация → формат → парсинг → валидация → обработчик → сериализация → лог
- **JSON-RPC 2.0 сервер** с реестром методов через автообнаружение модулей-`HTTPПодсистема`
- **Аутентификация** Basic / Bearer / APIKey (с справочником ключей и SHA-256 хешем)
- **Swagger-генератор** OpenAPI 3.0 из реестров методов и маршрутов + UI через CDN
- **Идемпотентность** по RFC 7807 (`Idempotency-Key`)
- **Журнал входящих запросов** в регистре сведений + штатный журнал регистрации
- **Регламентные задания** очистки журналов и разблокировки конфликтов
- **Самодиагностика** — обработка `http_СамодиагностикаHTTPСервера` с unit + e2e тестами и проверкой обновлений из этого же репо

## Регистрация прикладной HTTPПодсистемы

В любом расширении / основной конфигурации создаётся общий модуль с именем
оканчивающимся на `HTTPПодсистема`. В модуле:

```bsl
// МойСервисHTTPПодсистема

// Маркер: подтверждает что модуль предназначен для http_HTTPСервер.
Функция ЭтоHTTPПодсистема() Экспорт
    Возврат Истина;
КонецФункции

Процедура ПриЗаполненииМетодовJRPC(Методы) Экспорт
    Метод = Методы.Добавить();
    Метод.Имя        = "users.find";
    Метод.Обработчик = "МойСервисJRPC.НайтиПользователя";
    Метод.ТипВызова  = "Функция";
КонецПроцедуры
```

Фреймворк сам найдёт модуль через перечисление `Метаданные.ОбщиеМодули`, прочитает методы и диспетчеризует JSON-RPC-вызовы.

## Установка

1. Скачать `.cfe` из [Releases](../../releases)
2. Конфигуратор → Конфигурация → Расширения конфигурации → **Добавить из файла…**
3. Обновить конфигурацию БД (F7)

## Эндпоинты

| Маршрут | Описание |
|---|---|
| `GET /<base>/hs/http/ping` | Health-check |
| `POST /<base>/hs/jrpc/call` | JSON-RPC 2.0 endpoint |
| `GET /<base>/hs/swagger/swagger.json` | OpenAPI 3.0 spec |
| `GET /<base>/hs/swagger/` | Swagger UI (CDN) |
